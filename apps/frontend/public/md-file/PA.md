### 一、我们收集哪些信息

在你使用本服务的过程中，我们仅在实现特定目的所必需的范围内收集和处理一定数量的信息。根据信息的来源和性质，主要包括以下几类：

#### 1. 你在使用 Discord 登录时提供或授权的信息

当你选择使用「Discord 登录」功能以访问本网站时，我们会通过 Discord 提供的 OAuth 授权机制获取你在授权页面上明确同意提供的基本账户资料。具体包括但不限于：

- Discord 用户 ID；
- 用户名 / 昵称（可能包含你的 Discord 标签、显示名等）；
- 头像链接或相关头像信息；
- 其他在 Discord OAuth 授权页面中列明并经你同意授权的资料。

上述信息的主要用途在于：

- 在本网站内唯一识别和区分用户（本网站的账号体系是基于你的 Discord 账号构建的）；
- 将你的使用记录（例如设置偏好）与特定身份相对应，以便在你再次登录时为你恢复个性化的使用体验；
- 在必要时与 Discord 账号信息进行匹配，以协助安全审计或滥用行为处理（例如封禁或限制特定账号的访问）。

我们不会通过 Discord 获取超出授权范围之外的资料，也不会尝试在未经你授权的情况下访问你的私聊、服务器列表、消息内容等与本网站无关的数据。

#### 2. 使用本网站过程中由服务器端保存的信息

为提供、维持与改进本服务的基本功能，以及保障服务在技术层面的正常运行，我们会在自有服务器上保存以下有限的数据类型：

- **会话 key / 会话标识（session key 或类似令牌）**：
  当你通过 Discord 登录后，我们会为你生成或记录一个与该登录状态相关联的会话标识，用于：
  - 维持你的登录状态（例如在多页浏览时不需要重复登录）；
  - 识别连续请求是否来自同一用户或同一登录会话；
  - 实现对登录态的基本安全控制（例如在注销或过期后失效）。

- **用户偏好设置（user preferences）**：
  本网站可能允许你对部分功能进行个性化设置，例如但不限于：
  - 内容显示风格与布局选项；
  - 搜索过滤条件、排序偏好或其他与使用体验相关的设置。

  为便于在不同设备或不同时间点为你提供一致的体验，这些用户偏好设置会与您的 Discord 身份关联并保存到服务器端。在你下次登录时，我们会根据这些记录自动为你还原相关配置。

  你可在本网站的「设置」或类似页面中，对上述偏好进行修改、重置或删除。当你选择删除或重置相关偏好设置时，我们将在技术可行范围内尽快在服务器端同步删除或覆盖相应的数据记录，以确保这些偏好不再用于后续会话。

我们在收集和处理上述信息时，遵循数据最小化和目的限制原则，即仅保存实现相关功能所必需的最少量数据，不额外扩展用途。

#### 3. 本地保存的信息（不上传到服务器）

为提升你在本地设备上的使用便利性和响应速度，我们会使用浏览器提供的本地存储机制（包括但不限于 localStorage、sessionStorage、IndexedDB 或缓存等）保存一部分与个体设备强相关的使用记录。这类信息包括但不限于：

- **浏览历史和搜索历史**：
  - 浏览历史和搜索历史仅保存在你所使用的本地设备中，例如浏览器本地存储或缓存空间；
  - 该等信息不会自动上传到我们的服务器，也不在我们的服务器数据库中持久化保存；
  - 因此，从技术实现上，我们在服务器端**无法查看、无法远程读取、无法长期存储或系统性分析**你的本地浏览历史和搜索历史。

你可以随时通过浏览器自带的「清除浏览数据」「清除站点数据」等功能，或通过浏览器的隐私/无痕模式，自主地管理和删除这些仅存于本地的记录。删除后，这些数据将不再对你后续使用产生影响。

#### 4. 自动产生的技术性信息（通过 Cloudflare 等第三方产生的网络日志）

在网络传输与访问过程中，为了保证本网站的可访问性、安全性以及基础性能，我们会使用第三方内容分发网络（CDN）和反向代理服务，例如 Cloudflare。这意味着：

- 你的访问请求在到达我们自有服务器之前，会先经过 Cloudflare 等第三方基础设施；
- 在这一过程中，与网络访问相关的一些技术性信息（例如 IP 地址、User-Agent、请求的 URL、访问时间戳、协议信息等）会由 Cloudflare 按照其自有的隐私政策和日志策略进行记录、处理和临时或中期存储。

需要特别说明的是：

- **我们的自有应用服务器本身不会记录或长期保存**上述 IP 地址、User-Agent、浏览器类型、通讯协议标识等访问日志信息；
- 我们仅可能在安全防护、性能监测、故障排查或滥用行为处置的合理范围内，通过 Cloudflare 提供的面板或工具，查看这些由 Cloudflare 维护的日志或统计信息；
- 这类技术性数据主要用于识别攻击模式、防御 DDoS、排查连接异常以及进行总体访问量统计，不会被我们用于个性化广告投放或针对性画像分析。

1. **提供和维护本服务**

   - 使用 Discord 登录信息识别和验证你的身份，从而建立和维护与你的持续会话；
   - 利用 session key 或等效标识来维持你的登录状态，避免你在网站内频繁重复认证；
   - 保存并应用与你的账号绑定的用户偏好设置，在你每次登录时自动恢复你的自定义配置。

   这些处理行为通常基于 GDPR 第 6 条第 1 款 b 项（为履行你与我们之间的类似合同关系 / 提供你请求的服务所必需）以及在某些情形下结合第 6 条第 1 款 f 项（我们的合法利益）作为法律基础。

2. **保障网站安全与防止滥用**

   为确保本网站及其基础设施的可用性、安全性与完整性，我们可能：

   - 使用 Cloudflare 提供的安全与防护功能，识别和阻断恶意流量、自动化攻击、滥用接口调用等行为；
   - 结合有限的技术性数据（例如访问频率、错误返回码等统计信息），对可能影响整个服务稳定性的异常行为进行分析和处理；
   - 在出现系统错误、性能问题或稳定性风险时，通过 Cloudflare 的工具或摘要信息协助故障定位。

   上述处理通常基于 GDPR 第 6 条第 1 款 f 项（出于我们维护网络和信息安全的合法利益）进行。

3. **遵守法律义务**

   在某些情况下，我们可能在德国或欧盟适用法律的要求下：

   - 依据法院裁定、司法请求或有权监管机构的合法要求，协助提供必要的信息；
   - 履行与数据保留、审计或合规相关的强制义务。

   在此类场景下，处理行为通常基于 GDPR 第 6 条第 1 款 c 项（履行法律义务）作为法律基础。

我们承诺不会将你的数据用于与上述目的不兼容的其他用途，如需变更处理目的，我们将在法律要求的范围内另行征得你的同意或提供补充说明。

### 四、数据存储地点与保存期限

#### 1. 数据存储地点

- 我们自有的应用服务器托管于德国境内，用户偏好设置、会话 key 等数据通常存储于位于德国或其他欧盟成员国的服务器或数据库中；
- 我们使用包括 Cloudflare 在内的第三方网络基础设施作为内容分发网络（CDN）和反向代理，以提升访问速度、增强可用性并提供防护机制。在这种技术架构下：
  - 你的网络请求将先经过 Cloudflare 的边缘节点或相关服务器，之后再被转发至我们在德国的主服务器；
  - 与网络传输相关的部分技术性信息（例如 IP 地址、User-Agent、请求 URL、时间戳、协议信息等）将由 Cloudflare 依据其自身的隐私政策和日志策略进行记录、处理和保存；
  - 我们自有的应用服务器**不会另外记录或长期保存 IP 地址、User-Agent、浏览器类型、通讯协议等访问日志信息**，也不会在本地建立与之对应的长期日志归档。

#### 2. 保存期限

在确定各类数据的保存期限时，我们遵循 GDPR 所确立的「存储期限限制原则」，即在实现收集目的所必需的最短期限内保存数据，并在该目的达成或不再适用时，删除或匿名化相关数据。具体而言：

- **会话 key / 会话标识（session key）**：
  - 用于维持你的登录状态和会话连续性；
  - 一般将在会话到期、你主动登出、或系统认定会话长期未活动之后，在系统预设的短期内自动失效或被删除；
  - 具体失效时间可能取决于我们的技术配置与安全策略（例如以小时或天为单位），其设计目标是：在保证合理使用体验的前提下，避免会话长期处于未授权或无人使用状态。

- **用户偏好设置（user preferences）**：
  - 为了在你多次登录之间保持一致的界面与体验，我们会在你的账号保持活跃状态期间保留这些偏好设置；
  - 当你在设置页面中主动删除或重置某些偏好时，我们会尽可能在短时间内在服务器端删除、重置或匿名化这些数据，使其不再影响后续的会话；
  - 如你的账号长期处于不活跃状态，我们可能在合理的期限后对部分与账号关联的偏好数据进行清理或归档，具体策略视实际运营与技术条件调整。

- **Cloudflare 侧的访问日志**：
  - 与 IP 地址、User-Agent、请求路径、时间戳、协议版本等相关的访问日志主要由 Cloudflare 依法且依其隐私政策进行记录和保管；
  - Cloudflare 对数据的具体保留期限、日志轮转策略与数据处理方式由其自行决定，可能根据服务类型、安全策略及法律要求进行调整；
  - 我们一般仅在安全防护、性能调优、故障排查、滥用防控的必要范围内，间接地通过 Cloudflare 提供的界面查看聚合或局部日志，不会将完整的原始日志持续下载并在我们自有服务器上长期存储。

当某项数据已不再为实现既定目的所必需，且不存在法律规定的强制保留义务时，我们会在合理时间内删除、匿名化或以其他方式对其进行处理。对于由 Cloudflare 记录和保存的数据，则以 Cloudflare 的保留与删除策略为准。

### 五、你的权利（GDPR 下的数据主体权利）

在适用的法律框架下，尤其是在 GDPR 的约束下，你作为数据主体，对我们所处理的与你相关的个人数据享有一系列权利。在满足相应条件的情况下，你可以行使包括但不限于以下权利：

1. **访问权（Right of Access）**
   你有权请求确认我们是否正在处理与你相关的个人数据，并在合法范围内获得该等数据的副本，以及关于数据来源、处理目的、数据类别、接收方或接收方类别、预期保存期限等信息。

2. **更正权（Right to Rectification）**
   如你发现我们持有的关于你的个人数据存在不准确、不完整或已过时的情形，你有权要求我们及时对相关数据进行更正或补充，以确保数据在合理范围内保持准确性和完整性。

3. **删除权 / 被遗忘权（Right to Erasure）**
   在特定条件下（例如：数据已不再为收集目的所必需；你撤回基于同意的处理且不存在其他法律依据；你依法反对处理且无优先合法理由继续处理；数据被违法处理等），你可以要求我们删除与你相关的个人数据。我们将在法律要求和技术可行范围内进行评估并作出相应处理。

4. **限制处理权（Right to Restriction of Processing）**
   在某些情况下（例如你质疑数据的准确性、数据处理违法但你不希望删除、我们不再需要该数据但你需要其用于主张、行使或抗辩法律请求等），你可以要求我们暂时限制对你数据的进一步处理。在限制期间，除了存储之外，我们原则上不得对这些数据进行其他处理，除非你同意或出于特定法律需要。

5. **数据可携带权（Right to Data Portability）**
   在特定情形下（尤其是数据处理基于同意或合同，且通过自动化手段实现时），你有权以结构化、通用且机器可读的格式获取你向我们提供的与自身相关的个人数据，并在技术可行的范围内要求我们将其直接传输给你指定的其他数据控制者。

6. **反对权（Right to Object）**
   当我们以「合法利益」（GDPR 第 6 条第 1 款 f 项）为基础处理你的数据时，你可以基于与你特殊情况相关的理由，随时对该类处理提出反对。在接到你的反对请求后，我们将评估是否存在优先于你利益、权利和自由的重大合法理由继续处理，或该数据是否因确立、行使或抗辩法律请求需要而必须继续处理，否则我们通常将停止相关处理活动。

7. **撤回同意的权利（Right to Withdraw Consent）**
   若我们对某些处理活动是基于你的明确同意（GDPR 第 6 条第 1 款 a 项），你有权随时撤回该同意。撤回应当自生效之时起发生效力，对撤回前在同意基础上进行的处理不产生追溯影响。

如你希望行使上述任何权利，请通过本隐私说明末尾的联系方式与我们取得联系。为了保护你的数据安全、避免他人冒充，我们可能会在合理范围内要求你提供必要的补充信息以核实你的身份。

此外，你有权向你所在欧盟成员国或德国的数据保护监管机构提出投诉。如果你在德国境内，你可以选择向德国联邦数据保护与信息自由专员（BfDI）或你所在联邦州的数据保护监管机构提出申诉。

   为实现本网站的基本功能并提升用户体验，我们可能会在你的浏览器中使用以下技术：

   这些技术主要用于：

   - 管理和维持登录状态（例如记录你是否已经通过 Discord 完成认证）；

2. **管理与删除 Cookies / 本地存储**

   你可以通过浏览器的设置界面查看、管理和删除本网站设置的 Cookies 以及本地存储数据。不同浏览器（如 Chrome、Firefox、Safari、Edge 等）可能提供不同的管理路径和操作方式，你可以参考其官方帮助文档进行操作。你也可以使用浏览器的「隐私模式」或「无痕模式」来减少本地数据的长期保存。
